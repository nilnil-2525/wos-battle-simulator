# WOS戦闘シミュレーター リリースノート (V43)
**リリース日: 2026-06-19**

本バージョン（V43）では、戦闘シミュレーションの利便性を向上させる**「プリセット機能（保存・読込）」の新規実装**に加え、目に優しい**「ターミナルグリーンテーマ」への刷新**、およびUI全体のデザインシステムの大幅な近代化・近代英語表記への統一を実施。

---

## 🛠 変更・修正詳細

### 1. プリセット保存・読込機能（1〜5スロット）の新規実装
* **対象箇所**: [App.jsx](./src/App.jsx) / [ArmyPanel.jsx](./src/components/ArmyPanel.jsx) / [styles.css](./src/styles.css)
* **対応内容**:
  - **英雄編成プリセット**: 盾・槍・弓のリーダーおよび乗り手4名の編成スロット（1〜5）の保存・ロードに対応。
  - **兵種構成プリセット**: 各兵種（盾・槍・弓）のTier、兵士数、およびバフ（％）スロット（1〜5）の保存・ロードに対応。
  - **個別スロットクリア（消去）機能**: `Preset 💾`（登録モード）の際、データがあるスロットの右上に赤丸の削除ボタン（`×`）を配置し、誤操作防止の確認ダイアログを経て個別に初期化できる機能を追加。
  - **セキュア設計**: LocalStorageからパースする際の例外ハンドリングを追加し、破損データによるクラッシュを防止。

### 2. 「目に優しいターミナルグリーン」テーマの導入と初期ロードのチラつき防止
* **対象箇所**: [index.html](./index.html) / [App.jsx](./src/App.jsx) / [styles.css](./src/styles.css)
* **対応内容**:
  - 従来のライト、ダーク、氷風テーマに加え、サイバー感のある「ターミナルグリーン（Terminal Green）」テーマを追加。
  - 初回アクセス時やリロード時にテーマ切り替えが遅れて発生する白いチラつき（FOUC）を防ぐため、インラインでテーマを適用するプリロードスクリプトを `index.html` に導入。

### 3. アプリタイトルの刷新とデザインのアップグレード
* **対象箇所**: [App.jsx](./src/App.jsx) / [styles.css](./src/styles.css)
* **対応内容**:
  - アプリタイトルのスペルミスを修正（`Sumilator` ➔ `Simulator`）。
  - レトロゲーム風のピクセルフォント（`Press Start 2P`）をGoogle Fontsからインポートし、3色グラデーションとグローシャドウを適用してプレミアム感を向上。

### 4. 両サイド部隊パネルのタイトルデザイン統一
* **対象箇所**: [ArmyPanel.jsx](./src/components/ArmyPanel.jsx) / [styles.css](./src/styles.css)
* **対応内容**:
  - 部隊パネルのヘッダーを「味方 ALLY」「敵 ENEMY」の英語併記に変更し、それぞれの勢力カラーを基調とした薄い下線付きのレイアウトに統一。
  - 総兵力表示には等幅フォント（`font-mono`）を適用し、桁がズレても綺麗に整列するように対応。

### 5. スキル発動回数セクションの刷新とマルチカラー化
* **対象箇所**: [App.jsx](./src/App.jsx) / [styles.css](./src/styles.css)
* **対応内容**:
  - 味方と敵のスキル発動回数を左右に分割したカードリスト構造に刷新。未発動時のプレースホルダー表示を追加。
  - 発動回数の数値を兵種・役割（盾・槍・弓・首領・その他）に応じた鮮やかなマルチカラー配色に変更（ライト/ダーク/氷風/ターミナルの各テーマで最適な視認性を持つ色を選定）し、戦況を一目で把握しやすく改善。

### 6. 兵種ステータス値と残存兵力の1行表示における詰まり解消
* **対象箇所**: [ArmyPanel.jsx](./src/components/ArmyPanel.jsx)
* **対応内容**:
  - 各兵種のステータス（攻/防/殺/HP）と残存兵力の1行表示において、文字サイズを `text-xs` に引き上げ、コロンの後に適切なスペースを配置、さらに項目間ギャップを `gap-3` に広げるなどして窮屈な詰まり感を解消し、可読性を大幅に向上。

### 7. モーダルウィンドウの調整と英語表記（Skill Editor / Skill List）への統一
* **対象箇所**: [App.jsx](./src/App.jsx) / [InlineSkillDictionary.jsx](./src/components/InlineSkillDictionary.jsx) / [SkillTestEditorModal.jsx](./src/components/SkillTestEditorModal.jsx)
* **対応内容**: 
  - ツールバー上のボタン表示をそれぞれ `Skill Editor`, `Skill List`（旧：スキル重複辞典）に変更。
  - 各モーダルウィンドウの最上部に、それぞれの用途やバフの加算・乗算の計算仕様を説明する、テーマ色と連動したインフォメーションボックス（注意書き改行付き）を追加。
  - Skill Editorのモーダルについて、内部レイアウト構造は維持したまま、配色（背景、ヘッダー、カード、入力欄）のみを重複カテゴリー辞典と同様のライト/ダークテーマ・氷風テーマに連動する配色に統一し、ダークモード時の眩しさを解消。

# WOS戦闘シミュレーター リリースノート (V42)
**リリース日: 2026-06-16**

本バージョン（V42）では、戦闘シミュレーション計算のバグ修正、ターゲット追従ロジックの改善、確率スキルの適用条件の厳密化を行い、ゲーム実機とのシミュレーション精度の向上を図る。

---

## 🛠 変更・修正詳細

### 1. バフ・デバフカテゴリ集計関数 (`sumBuffCategory`) の引数ズレ修正
* **対象箇所**: [battleSimulator.js](./src/utils/battleSimulator.js)
* **不具合内容**: 特定の兵種を狙ったデバフ効果（`OppDefenseDown1`, `OppDefenseDown2`）を集計する際、引数の引き渡し順にズレがあり、攻撃側・防御側の兵種情報が正しく判定関数に伝わらず、一部のデバフ効果が計算から漏れるバグが発生。
* **修正内容**: `calculateDamageSplit` から `sumBuffCategory` を呼び出す際、攻撃側兵種 (`atkType`) を第3引数、防御側兵種 (`defType`) を第4引数として正しく設定し、デバフ対象が正しく評価されるように修正。

### 2. 槍兵「奇襲」発動時のターゲット追従とバフ引き継ぎの修正
* **対象箇所**: [battleSimulator.js](./src/utils/battleSimulator.js)
* **不具合内容**: T11槍兵の「奇襲」が発動すると攻撃対象が「弓兵」に切り替わるが、攻撃後に付与されるバフ（偶数回攻撃時バフ `spear_even_attack_after` 等）の付与先ターゲット判定において、奇襲先の「弓兵」ではなく、元の優先ターゲット（盾兵）の情報を引き継いでしまう不整合が存在。
* **修正内容**: 槍兵が攻撃した際に「実際に攻撃したターゲット」を `allySpearActualTarget` / `enemySpearActualTarget` に保存する処理を追加。ターン終了時のバフ引き継ぎ処理（`updateSpearActionBuffs`）でこの実際のターゲットを正確に引き継ぐようにし、奇襲先の「弓兵」に対して正しくデバフ等が適用されるよう修正。

### 3. ミアⅠスキルの抽選と適用ロジックの厳密化
* **対象箇所**: [battleSimulator.js](./src/utils/battleSimulator.js)
* **不具合内容**: ミアⅠ（`mia_atk_prob_50`）の50%確率の被ダメ上昇デバフは各優先度フェーズの開始時に判定されるが、そのフェーズで行動可能な自軍/敵軍の部隊が存在するか（生存しているか）を考慮せずに無条件で抽選処理が走る記述となっていた。
* **修正内容**: 各優先度フェーズの開始時、そのフェーズで行動可能な自軍/敵軍の部隊（`atkTypeAlly` / `atkTypeEnemy`）が生存している場合のみ、厳密にミアⅠの抽選判定を行うように条件を追加。
 
### 4. UIの安全設計（Reactカスタム確認モーダルの導入）
* **対象箇所**: WOS_battlesim.html (削除済み)
* **変更点**: シミュレーションのリセット（初期化）時にブラウザ標準の同期的な `confirm()` ダイアログを呼び出していたのを廃止し、Reactで構築したカスタム確認モーダルを導入。UXの統一感の向上と、誤操作によるバトルの初期化防止（安全設計）を実現。
