# WOS戦闘シミュレーター (wos_battle_simulator)

スマホゲーム「Whiteout Survival (WOS)」の戦闘進行を再現・シミュレーションするための、Vite + Reactで構築されたWebアプリケーションである。
通常の戦闘進行ログの出力に加え、モンテカルロ法を用いた1,000回のシミュレーションによる勝率分布や残り兵力の期待値・ばらつきを視覚的に分析できる。

---

## 🚀 主な機能

* **部隊・バフ設定**:
  - 兵種（盾兵・槍兵・弓兵）のTier（T1〜T11）や初期兵数の設定。
  - 各兵種のステータスバフ（攻撃・防御・殺傷・HP）のパーセンテージ入力。
* **英雄編成とスキルエディタ**:
  - 盾リーダー、槍リーダー、弓リーダー、および最大4名の「乗り手」英雄の編成。
  - **スキルエディタ**: 英雄ごとのスキル効果量や発動条件（タイミング）、計算枠（カテゴリー）をリアルタイムに変更・テスト可能。
* **2つのシミュレーションモード**:
  - **1回テスト (詳細ログ)**: 優先度フェーズごとの攻撃、スキルの発動、バフの適用、撃破数の詳細な計算内訳をターンごとにコンソール風ログで出力。
  - **1000回テスト (モンテカルロ法)**: ランダム確率スキル（ミアの確率発動、T11兵士スキル等）を含めた戦闘を1,000回行い、勝率（ドーナツチャート）や残存兵力の分布（ヒストグラム、平均値、中央値、標準偏差）を可視化。
* **スキル重複カテゴリー辞典**:
  - 同一枠のバフ（加算）と異なる枠のバフ（乗算）の仕組みを整理し、重複カテゴリーを一覧で確認できる辞書機能を内蔵。

## 🛠 前提環境・起動方法

本プロジェクトは Node.js 環境で動作する。

### 1. 依存パッケージのインストール
リポジトリのルートディレクトリで以下を実行する。
```bash
npm install
```

### 2. 開発サーバーの起動 (Vite)
ローカルで開発用サーバーを立ち上げる。
```bash
npm run dev
```
起動後、ターミナルに表示されるローカルURL（通常は `http://localhost:5173`）にブラウザでアクセスすること。

## 🧪 テストの実行と追加方法 (テーブル駆動テスト)

戦闘ロジックのデグレ防止および動作検証のため、YAMLファイルで定義したテストケースを自動実行するテーブル駆動テストを導入している。プログラミングの知識がない設計者でも、YAMLファイルを変更するだけでテストケースを追加・検証できる。

### 1. テストの実行方法
以下のコマンドを実行することで、すべてのテストケースが実行される。
```bash
npm run test
```

### 2. テストケースの追加方法
[src/utils/battleSimulator.test-cases.yaml](file:///home/hiroki/wos_battle_simulator/src/utils/battleSimulator.test-cases.yaml) の末尾に新しいケースを追加する。

```yaml
- name: "【テスト名】分かりやすい説明を記載"
  ally:
    shield: { tier: 11, troops: 100000, attack: 0, lethality: 0, defense: 0, hp: 0 }
    spear: { tier: 11, troops: 50000, attack: 50, lethality: 50, defense: 0, hp: 0 }
    # 奇襲やミアIスキルを強制発動させたい場合は、以下のフラグを指定する
    # kisyu_active: true
    # mia_i_active: true
    # 偶数回バフを1ターン目から検証したい場合は以下を指定する
    # spearAttackCount: 1
    # 英雄を編成する場合は以下を指定する
    # heroes: { leaderSpear: "gordon" }
  enemy:
    shield: { tier: 11, troops: 100000, attack: 0, lethality: 0, defense: 0, hp: 0 }
  expected:
    # 期待する合計撃破数
    kills_to_enemy: 1200
```

* **期待値の自動決定方法**: 期待値の計算が難しい場合は、一時的に `kills_to_enemy: 0` などでテストを実行し、テスト結果で出力された `Received` の値を期待値として書き直すことで、簡単に正確なテストケースを作成できる。

## 📖 戦闘計算仕様と処理フロー

戦闘の進行およびダメージ・スキル判定の仕様は、以下のドキュメントにまとめられている。

*   **[戦闘計算仕様書 (calculation_logic_summary.md)](./calculation_logic_summary.md)**: ダメージ計算式、各英雄スキル・兵士スキルの詳細な効果量や適用枠（カテゴリー）を記載。
*   **[計算プロセスフロー図 (calculation_flow.md)](./calculation_flow.md)**: ターンおよび優先度フェーズ（盾 ➔ 槍 ➔ 弓）の進行、およびスキル評価のタイミングを Mermaid.js で視覚化。

### 「ターン」と「フェーズ」の定義
*   **ターン (Turn)**: 戦闘の全体進行単位（最大1000ターン）。ターン開始時にターン開始時スキル（ミアⅢ、グレッグⅠなど）や周期バフが判定される。
*   **フェーズ (Phase)**: 1ターン内における兵種（盾 ➔ 槍 ➔ 弓）の個別行動ステップ。同じフェーズ内で、味方と敵の同一兵種が同時に対象兵種へ攻撃する。フェーズ開始時にフェーズ開始時スキル（ミアⅠ、グレッグⅡ、T11槍奇襲など）が判定される。
*   **攻撃計算**: 個々の攻撃時（ダメージ計算）のタイミングで、攻撃時スキル（ミアⅡ、ソニヤⅢ、ゴードンⅠ、T11弓兵スキルなど）やT11盾/槍の確率防御スキルがリアルタイムに判定される。

## 📄 ファイル構成

* **`index.html` / `src/`**: Vite + React 開発用のソースコードおよびUIコンポーネント。
* **`src/utils/battleSimulator.js`**: 戦闘計算のコアロジック（Reactから独立した純粋な計算エンジン）。
* **`src/utils/battleSimulator.test.js`**: YAMLテストケースを自動パースして Vitest で実行するテストランナーコード。
* **`src/utils/battleSimulator.test-cases.yaml`**: 設計者が自由に編集・追加可能な、戦闘シミュレータのテストケース定義ファイル。
* **`public/game-data.yaml`**: 兵種の基礎ステータスや英雄スキルの基礎値を管理するデータファイル。
* **`calculation_logic_summary.md`**: 戦闘進行、ダメージ計算式、スキル発動タイミングなどの詳細な仕様書。
* **`calculation_flow.md`**: `processOneTurn` と `calculateDamageSplit` の関係性、ターン/フェーズの概念、およびスキル抽選・ダメージ計算のプロセスを示す Mermaid.js フロー図。
* **`RELEASE_NOTE.md`**: 各バージョンの変更点（バグ修正や仕様更新）を記録したリリースノート。
* **`agents.md` / `skills.md`**: 開発を担当するAIエージェントの役割定義および技術スキル一覧。
